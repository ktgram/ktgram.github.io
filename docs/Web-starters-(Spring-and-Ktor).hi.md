### स्प्रिंग स्टार्टर

लाइब्रेरी के लिए स्प्रिंग स्टार्टर मॉड्यूल एक ऑटो-कॉन्फ़िगरेशन मॉड्यूल है जो स्प्रिंग बूट एप्लिकेशन में टेलीग्राम बॉट कार्यक्षमताओं को एकीकृत करता है। यह स्प्रिंग बूट के डिपेंडेंसी इंजेक्शन और कॉन्फ़िगरेशन प्रॉपर्टीज़ की शक्ति का लाभ उठाता है ताकि प्रदान की गई कॉन्फ़िगरेशन के आधार पर टेलीग्राम बॉट्स को स्वचालित रूप से कॉन्फ़िगर किया जा सके। यह लाइब्रेरी विशेष रूप से उन डेवलपर्स के लिए उपयोगी है जो कोटलिन और स्प्रिंग बूट का उपयोग करके टेलीग्राम बॉट्स बनाना चाहते हैं, बॉट विकास और प्रबंधन के लिए एक सुव्यवस्थित दृष्टिकोण प्रदान करते हैं।

### मुख्य विशेषताएं

- **ऑटो-कॉन्फ़िगरेशन**: लाइब्रेरी प्रदान की गई कॉन्फ़िगरेशन प्रॉपर्टीज़ के आधार पर टेलीग्राम बॉट्स को स्वचालित रूप से कॉन्फ़िगर करती है, मैनुअल सेटअप की आवश्यकता को समाप्त करती है।
- **कॉन्फ़िगरेशन प्रॉपर्टीज़**: यह बॉट सेटिंग्स को आसानी से अनुकूलित करने के लिए कॉन्फ़िगरेशन प्रॉपर्टीज़ का समर्थन करता है, जैसे बॉट टोकन, पैकेज नाम और पहचानकर्ता।
- **स्प्रिंग एकीकरण**: स्प्रिंग इकोसिस्टम के साथ सहजता से एकीकृत होता है, बॉट इंस्टेंसेस को प्रबंधित करने के लिए स्प्रिंग के डिपेंडेंसी इंजेक्शन और एप्लिकेशन कंटेक्स्ट का उपयोग करता है।
- **कोरुटाइन सपोर्ट**: कोटलिन कोरुटाइन्स का उपयोग असिंक्रोनस बॉट ऑपरेशन्स के लिए करता है, दक्ष और गैर-अवरोधक निष्पादन सुनिश्चित करता है।

### आरंभ करना

स्प्रिंग स्टार्टर लाइब्रेरी का उपयोग करने के लिए टेलीग्राम बॉट्स के लिए, आपको इसे अपने स्प्रिंग बूट प्रोजेक्ट में निर्भरता के रूप में शामिल करने की आवश्यकता है। लाइब्रेरी स्प्रिंग बूट एप्लिकेशन के साथ काम करने के लिए डिज़ाइन की गई है और कार्य करने के लिए स्प्रिंग बूट फ्रेमवर्क की आवश्यकता होती है।

#### निर्भरता

अपने `build.gradle` या `pom.xml` फ़ाइल में निम्नलिखित निर्भरता जोड़ें:

```gradle
dependencies {
    implementation 'eu.vendeli:spring-starter:<version>'
}
```

`<version>` को लाइब्रेरी का नवीनतम संस्करण से बदलें।

#### कॉन्फ़िगरेशन

लाइब्रेरी स्प्रिंग बूट के `@ConfigurationProperties` का उपयोग कॉन्फ़िगरेशन प्रॉपर्टीज़ को बाइंड करने के लिए करती है। आप अपने स्प्रिंग बूट एप्लिकेशन की `application.properties` या `application.yml` फ़ाइल में अपनी बॉट कॉन्फ़िगरेशन परिभाषित कर सकते हैं।

```yaml
ktgram:
 autoStartPolling: true
 shareHttpClient: true
 bot:
    - token: YOUR_BOT_TOKEN
      pckg: com.example.bot
      identifier: MyBot
```

#### उपयोग

एक बार लाइब्रेरी शामिल और कॉन्फ़िगर हो जाने पर, यह प्रदान की गई कॉन्फ़िगरेशन के आधार पर टेलीग्राम बॉट इंस्टेंसेस को स्वचालित रूप से बनाता और कॉन्फ़िगर करता है।

यह कई बॉट इंस्टेंसेस का भी समर्थन करता है, कई को प्रारंभ करने के लिए बस बॉट अनुभाग में नए प्रविष्टि के रूप में घोषित करें:

```yaml
ktgram:
 bot:
    - token: YOUR_BOT_TOKEN
    - token: SECOND_BOT_TOKEN
```

### उन्नत कॉन्फ़िगरेशन

अधिक उन्नत कॉन्फ़िगरेशन के लिए, जैसे बॉट व्यवहार को अनुकूलित करना या अन्य स्प्रिंग घटकों के साथ एकीकरण, आप `BotConfiguration` क्लास का विस्तार कर सकते हैं और इसके `applyCfg` विधि के माध्यम से बॉट कॉन्फ़िगरेशन बदल सकते हैं, आप [वहां](https://github.com/vendelieu/telegram-bot_template/blob/spring-bot/src/main/kotlin/com/example/springbot/configuration/BotConfig.kt) उदाहरण देख सकते हैं।

> [!टिप]
> प्रत्येक प्रारंभित इंस्टेंस को कस्टम कॉन्फ़िगरेशन के साथ कॉन्फ़िगर करने के लिए, उन्हें उनकी पहचानकर्ता द्वारा अलग करें (BotConfiguration क्लास में भी एक पहचानकर्ता है)।

### कोटलिन

मॉड्यूल टेलीग्राम बॉट्स के लिए एक वेबहुक सर्वर बनाने में सुविधा प्रदान करने के लिए डिज़ाइन किया गया है। यह डेवलपर्स को सर्वर को कॉन्फ़िगर करने की अनुमति देता है, जिसमें SSL/TLS सेटिंग्स शामिल हैं, और कस्टम कॉन्फ़िगरेशन के साथ कई टेलीग्राम बॉट्स घोषित करता है। सेटअप प्रक्रिया लचीली है, जिससे डेवलपर्स सर्वर को अपनी विशिष्ट आवश्यकताओं के अनुसार अनुकूलित कर सकते हैं।

### स्थापना

कोटलिन स्टार्टर स्थापित करने के लिए मुख्य निर्भरता में अतिरिक्त जोड़ें:

```gradle
dependencies {
    implementation("eu.vendeli:ktor-starter:x.y.z") // वहां
    // x.y.z को वर्तमान लाइब्रेरी संस्करण से बदलें
}
```

### मुख्य घटक

`serveWebhook` फ़ंक्शन

serveWebhook फ़ंक्शन लाइब्रेरी का मूल है। यह टेलीग्राम बॉट्स के लिए वेबहुक सर्वर सेट करता और शुरू करता है। यह दो पैरामीटर स्वीकार करता है:

- `wait`: एक बूलियन जो इंगित करता है कि क्या सर्वर को बंद होने से पहले एप्लिकेशन के रुकने की प्रतीक्षा करनी चाहिए। डिफ़ॉल्ट रूप से सत्य।
- `serverBuilder`: सर्वर को कॉन्फ़िगर करने वाला लैम्ब्डा फ़ंक्शन। डिफ़ॉल्ट रूप से एक खाली लैम्ब्डा।

### कॉन्फ़िगरेशन

* `WEBHOOK_PREFIX`: यह वेबहुक लिसनर रूट के लिए पते के पूर्वसर्ग के रूप में उपयोग किया जाने वाला पैरामीटर है। (डिफ़ॉल्ट रूप से "/")

#### सर्वर सेटअप

- `server`: EnvConfiguration या ManualConfiguration का उपयोग करके सर्वर कॉन्फ़िगरेशन सेट करने की एक विधि।
- `engine`: नेटी एप्लिकेशन इंजन को कॉन्फ़िगर करने की एक विधि।
- `ktorModule`: एप्लिकेशन में कोटलिन मॉड्यूल जोड़ने की एक विधि।

लाइब्रेरी सर्वर के लिए व्यापक रेंज के कॉन्फ़िगर करने योग्य पैरामीटर प्रदान करती है, जिसमें होस्ट, पोर्ट, SSL सेटिंग्स और अधिक शामिल हैं। इसके कॉन्फ़िगर करने के लिए दो ठोस विकल्प हैं:

* `EnvConfiguration`: `KTGRAM_` पूर्वसर्ग के साथ वातावरण से कॉन्फ़िगरेशन मान पढ़ता है।
* `ManualConfiguration`: कॉन्फ़िगरेशन मानों को मैन्युअल रूप से सेट करने की अनुमति देता है, `server {}` फ़ंक्शन में अपने पैरामीटर सेट करें।

सेट किए जा सकने वाले पैरामीटर की सूची है:

- `HOST`: सर्वर का होस्टनाम या IP पता।
- `PORT`: सर्वर के लिए पोर्ट नंबर।
- `SSL_PORT`: SSL/TLS कनेक्शनों के लिए पोर्ट नंबर।
- `PEM_PRIVATE_KEY_PATH`: PEM निजी कुंजी फ़ाइल का पथ।
- `PEM_CHAIN_PATH`: PEM प्रमाणपत्र श्रृंखला फ़ाइल का पथ।
- `PEM_PRIVATE_KEY`: PEM निजी कुंजी के रूप में एक वर्ण सरणि के रूप में पासवर्ड।
- `KEYSTORE_PATH`: Java KeyStore फ़ाइल का पथ।
- `KEYSTORE_PASSWORD`: KeyStore के लिए पासवर्ड।
- `KEY_ALIAS`: KeyStore में कुंजी के लिए उपनाम।
- `SSL_ON`: एक बूलियन जो इंगित करता है कि क्या SSL/TLS सक्षम होना चाहिए। डिफ़ॉल्ट रूप से सत्य।

> [!टिप]
> यदि पीईएम प्रमाणपत्र मौजूद हैं, तो मॉड्यूल स्वयं उन्हें निर्दिष्ट पथ पर jks स्टोरेज से बनाएगा।

#### बॉट कॉन्फ़िगरेशन:

बॉट को कॉन्फ़िगर करने के लिए `declareBot {}` कॉल करें जिसमें ऐसे पैरामीटर हैं:

- `token`: बॉट टोकन।
- `pckg`: बॉट के लिए पैकेज नाम।
- `configuration`: बॉट को कॉन्फ़िगर करने वाला लैम्ब्डा फ़ंक्शन।
- `handlingBehaviour`: बॉट के हैंडलिंग व्यवहार को सेट करने वाला लैम्ब्डा फ़ंक्शन।
- `onInit`: बॉट प्रारंभ होने पर निष्पादित किया जाने वाला लैम्ब्डा फ़ंक्शन।

### उदाहरण उपयोग

इस मॉड्यूल का उपयोग करने के लिए, `serveWebhook` फ़ंक्शन कॉल करें, अपनी वांछित सेटिंग्स के साथ इसे कॉन्फ़िगर करें, अपने बॉट्स घोषित करें। यहां एक सरलीकृत उदाहरण है:

```kotlin
fun main() = runBlocking {
    serveWebhook {
        server {
            HOST = "0.0.0.0"
            PORT = 8080
            SSL_PORT = 8443

            PEM_PRIVATE_KEY_PATH = "/etc/letsencrypt/live/example.com/privkey.pem"
            PEM_CHAIN_PATH = "/etc/letsencrypt/live/example.com/fullchain.pem"
            PEM_PRIVATE_KEY = "pem_changeit".toCharArray()

            KEYSTORE_PATH = "/etc/ssl/certs/java/cacerts/bot_keystore.jks"
            KEYSTORE_PASSWORD = "changeit".toCharArray()
            // आवश्यकतानुसार अन्य कॉन्फ़िगरेशन पैरामीटर सेट करें
        }
        declareBot {
            token = "YOUR_BOT_TOKEN"
            // अन्य बॉट सेटिंग्स कॉन्फ़िगर करें
        }
        // आवश्यकतानुसार अधिक बॉट्स जोड़ें या अन्य पैरामीटर सेट करें
    }
}
```

> [!सावधानी]
> सब कुछ काम करने के लिए वेबहुक सेट करना न भूलें। :)

डिफ़ॉल्ट रूप से मॉड्यूल वेबहुक लिसनिंग एंडपॉइंट्स को `host/BOT_TOKEN` के रूप में सर्व करेगा
