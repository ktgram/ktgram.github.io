### Spring starter

ماژول Spring Starter برای کتابخانه یک ماژول خودکار-پیکره‌بندی است که قابلیت‌های ربات تلگرام را به برنامه‌های Spring Boot یکپارچه می‌کند. این ماژول از قدرت تزریق وابستگی‌های Spring Boot و ویژگی‌های پیکره‌بندی برای پیکره‌بندی خودکار ربات‌های تلگرام بر اساس پیکره‌بندی ارائه شده استفاده می‌کند. این کتابخانه به ویژه برای توسعه‌دهندگانی که می‌خواهند ربات‌های تلگرام با استفاده از Kotlin و Spring Boot بسازند، مفید است و رویکردی ساده‌گرا برای توسعه و مدیریت ربات ارائه می‌دهد.

### ویژگی‌های کلیدی

- **پیکره‌بندی خودکار**: کتابخانه بر اساس ویژگی‌های پیکره‌بندی ارائه شده، ربات‌های تلگرام را خودکار پیکره‌بندی می‌کند و نیازی به راه‌اندازی دستی را حذف می‌کند.
- **ویژگی‌های پیکره‌بندی**: پشتیبانی از ویژگی‌های پیکره‌بندی برای سفارشی‌سازی آسان تنظیمات ربات مانند توکن‌های ربات، نام بسته‌ها و شناسه‌ها ارائه می‌کند.
- **یکپارچگی Spring**: به‌صورت یکپارچه با اکوسیستم Spring ادغام می‌شود و از تزریق وابستگی‌های Spring و زمینه برنامه برای مدیریت نمونه‌های ربات استفاده می‌کند.
- **پشتیبانی Coroutine**: از coroutineهای Kotlin برای عملیات‌های ناهمزمان ربات استفاده می‌کند و اجرای کارآمد و غیرمسدود را تضمین می‌کند.

### شروع کار

برای استفاده از کتابخانه Spring Starter برای ربات‌های تلگرام، باید آن را به عنوان وابستگی در پروژه Spring Boot خود اضافه کنید. کتابخانه برای کار با برنامه‌های Spring Boot طراحی شده است و نیازمند چارچوب Spring Boot برای عملکرد دارد.

#### وابستگی

وابستگی زیر را به فایل `build.gradle` یا `pom.xml` خود اضافه کنید:

```gradle
dependencies {
    implementation 'eu.vendeli:spring-starter:<version>'
}
```

`برای <version>` با آخرین نسخه کتابخانه جایگزین کنید.

#### پیکره‌بندی

کتابخانه از `@ConfigurationProperties` Spring Boot برای اتصال ویژگی‌های پیکره‌بندی استفاده می‌کند. می‌توانید پیکره‌بندی ربات‌های خود را در فایل `application.properties` یا `application.yml` برنامه Spring Boot خود تعریف کنید.

```yaml
ktgram:
 autoStartPolling: true
 shareHttpClient: true
 bot:
    - token: YOUR_BOT_TOKEN
      pckg: com.example.bot
      identifier: MyBot
```

#### استفاده

پس از اینکه کتابخانه گنجانده شده و پیکره‌بندی شده است، به‌صورت خودکار نمونه‌های ربات تلگرام را ایجاد و پیکره‌بندی می‌کند بر اساس پیکره‌بندی ارائه شده.

همچنین از چند نمونه ربات پشتیبانی می‌کند، برای راه‌اندازی چندین نمونه فقط آن را به عنوان ورودی جدید در بخش bot اعلام کنید:

```yaml
ktgram:
 bot:
    - token: YOUR_BOT_TOKEN
    - token: SECOND_BOT_TOKEN
```

### پیکره‌بندی پیشرفته

برای پیکره‌بندی‌های پیشرفته‌تر مانند سفارشی‌سازی رفتار ربات یا ادغام با سایر کامپوننت‌های Spring، می‌توانید کلاس `BotConfiguration` را گسترش داده و پیکره‌بندی ربات را از طریق متد `applyCfg` آن تغییر دهید، می‌توانید نمونه‌ای [در اینجا](https://github.com/vendelieu/telegram-bot_template/blob/spring-bot/src/main/kotlin/com/example/springbot/configuration/BotConfig.kt) ببینید.

> [!TIP]
> برای پیکره‌بندی هر نمونه‌ای که مقداردهی شده با پیکره‌بندی سفارشی، آن‌ها را بر اساس شناسه خود (کلاس BotConfiguration نیز یک شناسه دارد) متمایز کنید.

### Ktor

ماژول برای تسهیل ایجاد سرور webhook برای ربات‌های تلگرام طراحی شده است. این ماژول به توسعه‌دهندگان اجازه می‌دهد سرور را پیکره‌بندی کنند، شامل تنظیمات SSL/TLS، و چندین ربات تلگرام با پیکره‌بندی‌های سفارشی اعلام کنند. فرآیند راه‌اندازی انعطاف‌پذیر است و به توسعه‌دهندگان اجازه می‌دهد سرور را به نیازهای خاص خود سفارشی کنند.

### نصب

برای نصب ktor starter به وابستگی اصلی اضافه کنید:

```gradle
dependencies {
    implementation("eu.vendeli:ktor-starter:x.y.z") // there
    // change x.y.z to current library version
}
```

### کامپوننت‌های کلیدی

تابع `serveWebhook`

تابع serveWebhook هسته کتابخانه است. این تابع سرور webhook را برای ربات‌های تلگرام پیکره‌بندی و راه‌اندازی می‌کند. دو پارامتر قبول می‌کند:

- `wait`: یک بولی که نشان می‌دهد آیا سرور باید قبل از خاموشی منتظر توقف برنامه بماند. به صورت پیش‌فرض درست است.
- `serverBuilder`: یک تابع لاندا برای پیکره‌بندی سرور. به صورت پیش‌فرض یک لاندا خالی است.

### پیکره‌بندی

* `WEBHOOK_PREFIX`: پارامتری است که برای پیشوند آدرس برای مسیر گوش‌دهنده webhook استفاده می‌شود. (به صورت پیش‌فرض "/")

#### راه‌اندازی سرور

- `server`: یک متد برای تنظیم پیکره‌بندی سرور با استفاده از EnvConfiguration یا ManualConfiguration.
- `engine`: یک متد برای پیکره‌بندی موتور برنامه Netty.
- `ktorModule`: یک متد برای اضافه کردن ماژول‌های Ktor به برنامه.

کتابخانه مجموعه گسترده‌ای از پارامترهای قابل تنظیم برای سرور ارائه می‌دهد، شامل میزبان، درگاه، تنظیمات SSL و غیره. دو گزینه عملی برای پیکره‌بندی آن وجود دارد:

* `EnvConfiguration`: مقادیر پیکره‌بندی را از محیط با پیشوند `KTGRAM_` می‌خواند.
* `ManualConfiguration`: اجازه تنظیم دستی مقادیر پیکره‌بندی را می‌دهد، پارامترهای خود را در تابع `server {}` تنظیم کنید.

فهرستی از پارامترهایی که می‌توان تنظیم کرد:

- `HOST`: نام میزبان یا آدرس IP سرور.
- `PORT`: شماره درگاه برای سرور.
- `SSL_PORT`: شماره درگاه برای اتصالات SSL/TLS.
- `PEM_PRIVATE_KEY_PATH`: مسیر به فایل کلید خصوصی PEM.
- `PEM_CHAIN_PATH`: مسیر به فایل زنجیره گواهی PEM.
- `PEM_PRIVATE_KEY`: رمز عبور کلید خصوصی PEM به عنوان آرایه کاراکتر.
- `KEYSTORE_PATH`: مسیر به فایل KeyStore جاوا.
- `KEYSTORE_PASSWORD`: رمز عبور برای KeyStore.
- `KEY_ALIAS`: نام مستعار برای کلید در KeyStore.
- `SSL_ON`: یک بولی که نشان می‌دهد آیا SSL/TLS باید فعال شود. به صورت پیش‌فرض درست است.

> [!TIP]
> اگر گواهی‌های pem موجود هستند، ماژول خودش ذخیره‌سازی jks را از آن‌ها در مسیر مشخص شده ایجاد می‌کند.

#### پیکره‌بندی ربات:

برای پیکره‌بندی ربات تابع `declareBot {}` را فراخوانی کنید که چنین پارامترهایی دارد:

- `token`: توکن ربات.
- `pckg`: نام بسته برای ربات.
- `configuration`: یک تابع لاندا برای پیکره‌بندی ربات.
- `handlingBehaviour`: یک تابع لاندا برای تنظیم رفتار مدیریت ربات.
- `onInit`: یک تابع لاندا که هنگام مقداردهی ربات اجرا می‌شود.

### نمونه استفاده

برای استفاده از این ماژول، تابع `serveWebhook` را فراخوانی کنید، آن را با تنظیمات دلخواه خود پیکره‌بندی کنید و ربات‌های خود را اعلام کنید. در اینجا یک مثال ساده‌سازی شده ارائه شده است:

```kotlin
fun main() = runBlocking {
    serveWebhook {
        server {
            HOST = "0.0.0.0"
            PORT = 8080
            SSL_PORT = 8443

            PEM_PRIVATE_KEY_PATH = "/etc/letsencrypt/live/example.com/privkey.pem"
            PEM_CHAIN_PATH = "/etc/letsencrypt/live/example.com/fullchain.pem"
            PEM_PRIVATE_KEY = "pem_changeit".toCharArray()

            KEYSTORE_PATH = "/etc/ssl/certs/java/cacerts/bot_keystore.jks"
            KEYSTORE_PASSWORD = "changeit".toCharArray()
            // Set other configuration parameters as needed
        }
        declareBot {
            token = "YOUR_BOT_TOKEN"
            // Configure other bot settings
        }
        // Add more bots or set other parameters if needed
    }
}
```

> [!CAUTION]
> فراموش نکنید webhook را تنظیم کنید تا همه چیز کار کند. :)

به صورت پیش‌فرض ماژول endpoints گوش‌دهنده webhook را به عنوان `host/BOT_TOKEN` ارائه می‌دهد
