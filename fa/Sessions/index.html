    Sessions

    SessionsВ¶

    Added in 9.5.

    ЫҢЪ© Session ШҜШіШӘЪҜЫҢШұЩҮвҖҢШ§ЫҢ ШЁШұШ§ЫҢ ЫҢЪ© ШЁШ®Шҙ Щ…ЩҶШ·ЩӮЫҢ ЪҜЩҒШӘвҖҢЩҲЪҜЩҲ Ш§ШіШӘ. ЩҮШұ ЩҫЫҢШ§Щ…ЫҢ Ъ©ЩҮ Ш§ШІ Ш·ШұЫҢЩӮ ШўЩҶ Ш№ШЁЩҲШұ Щ…ЫҢвҖҢЪ©ЩҶШҜ вҖ” ЩҮЩ… ЩҫЫҢШ§Щ…вҖҢЩҮШ§ЫҢ ЩҲШұЩҲШҜЫҢ Ъ©Ш§ШұШЁШұШ§ЩҶ ЩҲ ЩҮЩ… ЩҫЫҢШ§Щ…вҖҢЩҮШ§ЫҢ Ш®ШұЩҲШ¬ЫҢ ШұШЁШ§ШӘ вҖ” Ш«ШЁШӘ Щ…ЫҢвҖҢШҙЩҲШҜ ШӘШ§ ШұШЁШ§ШӘ ШЁШ№ШҜШ§ЩӢ ШЁШӘЩҲШ§ЩҶШҜ ШўЩҶвҖҢЩҮШ§ ШұШ§ ЩҫШ®Шҙ ЫҢШ§ bulkвҖ‘delete Ъ©ЩҶШҜ ШҜШұ ЫҢЪ© ШӘЩ…Ш§Ші.

    ШІЫҢШұШіЫҢШіШӘЩ… ЩҮЩ…ЩҲШ§ШұЩҮ ЩҒШ№Ш§Щ„ Ш§ШіШӘ ШЁШ§ ШӘЩҶШёЫҢЩ…Ш§ШӘ ЩҫЫҢШҙвҖҢЩҒШұШ¶ Щ…Ш№ЩӮЩҲЩ„. ШұШЁШ§ШӘвҖҢЩҮШ§ЫҢЫҢ Ъ©ЩҮ ЩҮШұЪҜШІ ШЁЩҮ ШіШҙЩҶвҖҢЩҮШ§ ШҜШіШӘ ЩҶЩ…ЫҢвҖҢШІЩҶЩҶШҜ ЩҮШІЫҢЩҶЩҮ ШЁЩҮвҖҢШұЩҲШІШұШіШ§ЩҶЫҢ ШөЩҒШұ (ЩҫШ§ЫҢЩҫЩ„Ш§ЫҢЩҶ Ш§ЫҢЩҶШӘШұШіЩҫШӘЩҲШұ ЩҮШұ ШІЩ…Ш§ЩҶ Ъ©ЩҮ ШіШҙЩҶ ШЁШ§ШІ ЩҶШЁШ§ШҙШҜШҢ Ъ©ЩҲШӘШ§ЩҮ Щ…ЫҢвҖҢШҙЩҲШҜ) ЩҫШұШҜШ§Ш®ШӘ Щ…ЫҢвҖҢЪ©ЩҶЩҶШҜ.

    flowchart LR
    A["bot.sessions.get(...)"] --> B[Auto-subscribe SessionKey]
    B --> C["SessionTrackingInterceptor<br/>auto-records Incoming"]
    A --> S["session-aware send / sendReturning<br/>auto-records Outgoing"]
    C --> D[SessionStorage backend]
    S --> D
    D --> E["session.messages() / forget() / clear()"]
    E --> F["session.close() вҶ’ unsubscribe + drop"]

    Quick startВ¶

    val session = bot.sessions.get(chatId = chat.id, userId = user.id)

// Both incoming and outgoing are tracked automatically вҖ” just send through the session.
with(session) {
    message { "Hi, what's up?" }.send(bot)        // auto-tracked as Outgoing
}

// later вҖ” wipe everything we sent and received in this slice
session.clear()

    ШұШҜЫҢШ§ШЁЫҢ ШЁЩҮвҖҢШөЩҲШұШӘ Ш®ЩҲШҜЪ©Ш§Шұ ШҜШұ ЩҮШұ ШҜЩҲ Ш¬ЩҮШӘ Ш§ЩҶШ¬Ш§Щ… Щ…ЫҢвҖҢШҙЩҲШҜ:

    1. ШіШҙЩҶвҖҢЩҮШ§ Ш§ШІ bot.sessions Ъ©ЩҮ ЫҢЪ© SessionManager ШҜШұ ЩҮШұ TelegramBot ШҜШұ ШҜШіШӘШұШі Ш§ШіШӘШҢ ШЁЩҮвҖҢШҜШіШӘ Щ…ЫҢвҖҢШўЫҢЩҶШҜ.
    2. ШЁЩҮвҖҢШұЩҲШІШұШіШ§ЩҶЫҢвҖҢЩҮШ§ЫҢ ЩҲШұЩҲШҜЫҢ ШӘЩҲШіШ· Ш§ЫҢЩҶШӘШұШіЩҫШӘЩҲШұ ЩҫШ§ЫҢЩҫЩ„Ш§ЫҢЩҶ ШЁШұШ§ЫҢ ЩҮШұ Ъ©Щ„ЫҢШҜЫҢ Ъ©ЩҮ ШіШҙЩҶ ШЁШ§ШІ ШҜШ§ШұШҜШҢ Ш«ШЁШӘ Щ…ЫҢвҖҢШҙЩҲШҜ.
    3. ЩҫЫҢШ§Щ…вҖҢЩҮШ§ЫҢ Ш®ШұЩҲШ¬ЫҢ ЩҮШұ ШІЩ…Ш§ЩҶ Ъ©ЩҮ Ш§ШұШіШ§Щ„ ШӯШ§ЩҲЫҢ ШіШҙЩҶ ШЁШ§ШҙШҜШҢ Ш«ШЁШӘ Щ…ЫҢвҖҢШҙЩҲЩҶШҜ вҖ” ЪҶЩҮ ШЁШ§ Ш№ШЁЩҲШұ ШөШұЫҢШӯ ШіШҙЩҶ (action.send(to, bot, session) / sendReturning(to, bot, session)) ЩҲ ЪҶЩҮ ШЁШ§ Ш§Ш¬ШұШ§ЫҢ Ш§ШұШіШ§Щ„ ШҜШ§Ш®Щ„ ШЁЩ„ЩҲЪ© with(session) { ... } (Ш§ЩҲШұЩ„ЩҲШҜЩҮШ§ЫҢ ЩҫШ§ШұШ§Щ…ШӘШұ ШІЩ…ЫҢЩҶЩҮ ШіШҙЩҶ ШұШ§ ШЁШұШ§ЫҢ ШҙЩ…Ш§ Ш№ШЁЩҲШұ Щ…ЫҢвҖҢШҜЩҮЩҶШҜ).

    ЩҒШұШ§Ш®ЩҲШ§ЩҶЫҢ ШҜШіШӘЫҢ session.track(message, Direction.Outgoing) ЩҒЩӮШ· ШІЩ…Ш§ЩҶЫҢ Щ„Ш§ШІЩ… Ш§ШіШӘ Ъ©ЩҮ ШЁШ®ЩҲШ§ЩҮЫҢШҜ Message Ш§ЫҢ Ъ©ЩҮ Ш§ШІ Ш·ШұЫҢЩӮ Ш§ШұШіШ§Щ„ ШўЪҜШ§ЩҮЫҢвҖҢШҜШ§Шұ Ш§ШІ ШіШҙЩҶ ШӘЩҲЩ„ЫҢШҜ ЩҶШҙШҜЩҮ Ш§ШіШӘШҢ Ш«ШЁШӘ Ъ©ЩҶЫҢШҜ (Щ…Ш«Щ„Ш§ЩӢ ЩҫЫҢШ§Щ… ШҜШұЫҢШ§ЩҒШӘвҖҢШҙШҜЩҮ Ш§ШІ Щ…ЩҶШЁШ№ ШҜЫҢЪҜШұ ЫҢШ§ ЩҫЫҢШ§Щ…ЫҢ Ъ©ЩҮ ЩӮШЁЩ„ Ш§ШІ ЩҲШ¬ЩҲШҜ ШіШҙЩҶ Ш§ШұШіШ§Щ„ ШҙШҜЩҮ).

    Session keys & strategiesВ¶

    ЫҢЪ© SessionKey ЫҢЪ© ШіШҙЩҶ Щ…ЩҶШ·ЩӮЫҢ ШұШ§ ШҙЩҶШ§ШіШ§ЫҢЫҢ Щ…ЫҢвҖҢЪ©ЩҶШҜ. Ш§ЫҢЩҶ ЫҢЪ© ЩҶЩҲШ№ sealed Ш§ШіШӘ:

    • SessionKey.Chat(chatId, qualifier?) вҖ” ШіШҙЩҶ ШіШұШ§ШіШұЫҢ ЪҶШӘ Ъ©ЩҮ ШЁШұШ§ЫҢ ЩҮЩ…ЩҮ Ш§Ш№Ш¶Ш§ЫҢ ЪҶШӘ Щ…ШҙШӘШұЪ© Ш§ШіШӘ.
    • SessionKey.ChatUser(chatId, userId, qualifier?) вҖ” ШіШҙЩҶ Щ…Ш®ШөЩҲШө Ъ©Ш§ШұШЁШұ ШҜШ§Ш®Щ„ ЫҢЪ© ЪҶШӘ.

    ЩҫШ§ШұШ§Щ…ШӘШұ Ш§Ш®ШӘЫҢШ§ШұЫҢ qualifier ШЁЩҮ ЪҶЩҶШҜ ШіШҙЩҶ Щ…ШіШӘЩӮЩ„ Ш§Ш¬Ш§ШІЩҮ Щ…ЫҢвҖҢШҜЩҮШҜ ШЁШұШ§ЫҢ ЩҮЩ…Ш§ЩҶ ЪҶШӘ/Ъ©Ш§ШұШЁШұ ЩҮЩ…ШІЩ…Ш§ЩҶ ЩҲШ¬ЩҲШҜ ШҜШ§ШҙШӘЩҮ ШЁШ§ШҙЩҶШҜ (Щ…Ш«Щ„Ш§ЩӢ "wizard" ЩҲ "support").

    ЫҢЪ© SessionKeyStrategy ШӘШ№ЫҢЫҢЩҶ Щ…ЫҢвҖҢЪ©ЩҶШҜ Ъ©ШҜШ§Щ… Ъ©Щ„ЫҢШҜ ШЁШұШ§ЫҢ ШЁЩҮвҖҢШұЩҲШІШұШіШ§ЩҶЫҢ ШҜШ§ШҜЩҮвҖҢШҙШҜЩҮ ШЁЩҮвҖҢЪ©Ш§Шұ ШұЩҲШҜ. ШіЩҮ Ш§ШіШӘШұШ§ШӘЪҳЫҢ ЩҫЫҢШҙвҖҢЩҒШұШ¶ ЩҲШ¬ЩҲШҜ ШҜШ§ШұШҜ:

    StrategyBehaviour
    SessionKeyStrategy.ChatUser (default)ChatUser(chat, user) ЩҲЩӮШӘЫҢ ЩҮШұ ШҜЩҲ Щ…ЩҲШ¬ЩҲШҜ ШЁШ§ШҙЩҶШҜШҢ ШҜШұ ШәЫҢШұШ§ЫҢЩҶШөЩҲШұШӘ Chat(chat).
    SessionKeyStrategy.ChatЩҮЩ…ЫҢШҙЩҮ Chat(chat). ШЁШұШ§ЫҢ ШұШЁШ§ШӘвҖҢЩҮШ§ЫҢ ЩҶЩҲШ№ broadcast ЩҲ Ъ©Ш§ЩҶШ§Щ„вҖҢЩҮШ§ Щ…ЩҒЫҢШҜ Ш§ШіШӘ.
    SessionKeyStrategy.AutoШҜШұ ЪҶШӘвҖҢЩҮШ§ЫҢ Ш®ШөЩҲШөЫҢ Chat(chat)ШҢ ШҜШұ ШіШ§ЫҢШұ Щ…ЩҲШ§ШұШҜ ChatUser(chat, user).

    SessionKeyStrategy ЫҢЪ© fun interface Ш§ШіШӘ вҖ” Щ…ЫҢвҖҢШӘЩҲШ§ЩҶЫҢШҜ ЫҢЪ© ЩҫЫҢШ§ШҜЩҮвҖҢШіШ§ШІЫҢ ШіЩҒШ§ШұШҙЫҢ ЩҒШұШ§ЩҮЩ… Ъ©ЩҶЫҢШҜ Ш§ЪҜШұ ШЁЩҮ ШӯЩҲШІЩҮвҖҢШЁЩҶШҜЫҢ Ш®Ш§ШөЫҢ (Щ…Ш«Щ„Ш§ЩӢ ШЁШұ ШӯШіШЁ Ш§ШӘШөШ§Щ„ ШӘШ¬Ш§ШұЫҢ ЫҢШ§ Щ…ЩҲШ¶ЩҲШ№) ЩҶЫҢШ§ШІ ШҜШ§ШұЫҢШҜ.

    Tracking directionВ¶

    ЩҮШұ ЩҲШұЩҲШҜЫҢ ШЁЩҮвҖҢШ№ЩҶЩҲШ§ЩҶ ЩҲШұЩҲШҜЫҢ ЫҢШ§ Ш®ШұЩҲШ¬ЫҢ Ш§ШІ Ш·ШұЫҢЩӮ Direction Ш«ШЁШӘ Щ…ЫҢвҖҢШҙЩҲШҜ:

    • Direction.Incoming вҖ” ШҜШұЫҢШ§ЩҒШӘвҖҢШҙШҜЩҮ Ш§ШІ Ъ©Ш§ШұШЁШұ/ЪҶШӘ. ШЁЩҮвҖҢШөЩҲШұШӘ Ш®ЩҲШҜЪ©Ш§Шұ ШӘЩҲШіШ· SessionTrackingInterceptor ШЁШұШ§ЫҢ ЩҮШұ Ъ©Щ„ЫҢШҜЫҢ Ъ©ЩҮ ШіШҙЩҶ ШЁШ§ШІ ШҜШ§ШұШҜШҢ Ш«ШЁШӘ Щ…ЫҢвҖҢШҙЩҲШҜ.
    • Direction.Outgoing вҖ” Ш§ШұШіШ§Щ„вҖҢШҙШҜЩҮ ШӘЩҲШіШ· ШұШЁШ§ШӘ. ШЁЩҮвҖҢШөЩҲШұШӘ Ш®ЩҲШҜЪ©Ш§Шұ ЩҮШұ ШІЩ…Ш§ЩҶ Ъ©ЩҮ Ш§ШұШіШ§Щ„ ШўЪҜШ§ЩҮЫҢвҖҢШҜШ§Шұ Ш§ШІ ШіШҙЩҶ ШЁШ§ШҙШҜШҢ Ш«ШЁШӘ Щ…ЫҢвҖҢШҙЩҲШҜ (ЫҢШ§ action.send(to, bot, session) Щ…ШіШӘЩӮЫҢЩ…ШҢ ЫҢШ§ ШҜШ§Ш®Щ„ with(session) { вҖҰ }).

    session.track(message, direction) ШЁЩҮвҖҢШ№ЩҶЩҲШ§ЩҶ ШұШ§ЩҮвҖҢШӯЩ„ ЩҫШҙШӘЫҢШЁШ§ЩҶ ШЁШұШ§ЫҢ Щ…ЩҲШ§ШұШҜ ЩҶШ§ШҜШұЫҢ Ъ©ЩҮ ЩҮЫҢЪҶвҖҢЫҢЪ© Ш§ШІ Щ…ШіЫҢШұЩҮШ§ Ш§Ш№Щ…Ш§Щ„ ЩҶЩ…ЫҢвҖҢШҙЩҲШҜ (Щ…Ш«Щ„Ш§ЩӢ ЩҫШұ Ъ©ШұШҜЩҶ ЫҢЪ© Message Ъ©ЩҮ Ш§ШІ Щ…ЩҶШЁШ№ ШҜЫҢЪҜШұЫҢ ШЁЩҮвҖҢШҜШіШӘ ШўЩ…ШҜЩҮ) Ш§ШіШӘЩҒШ§ШҜЩҮ Щ…ЫҢвҖҢШҙЩҲШҜ.

    ЩҲШұЩҲШҜЫҢвҖҢЩҮШ§ЫҢ Ш«ШЁШӘвҖҢШҙШҜЩҮ ШЁЩҮвҖҢШ№ЩҶЩҲШ§ЩҶ Щ…ЩӮШ§ШҜЫҢШұ TrackedMessage ШҙШ§Щ…Щ„ messageIdШҢ chatIdШҢ userId Ш§Ш®ШӘЫҢШ§ШұЫҢШҢ MessageKindШҢ Ш¬ЩҮШӘШҢ businessConnectionId Ш§Ш®ШӘЫҢШ§ШұЫҢ ЩҲ ШІЩ…Ш§ЩҶвҖҢЩ…ЩҮШұ Instant Ш°Ш®ЫҢШұЩҮ Щ…ЫҢвҖҢШҙЩҲЩҶШҜ.

    Session APIВ¶

    interface Session {
    val key: SessionKey
    val chatId: Long
    val userId: Long?
    val bot: TelegramBot

    suspend fun track(message: Message, direction: Direction = Direction.Outgoing)
    suspend fun track(update: ProcessedUpdate, direction: Direction = Direction.Incoming)
    suspend fun messages(): List<TrackedMessage>

    suspend fun clear(
        bot: TelegramBot = this.bot,
        predicate: (TrackedMessage) -> Boolean = { true },
    ): Int

    suspend fun forget(predicate: (TrackedMessage) -> Boolean = { true }): Int
    suspend fun close()
}
    • track ЫҢЪ© ЩҫЫҢШ§Щ… ШұШ§ Ш«ШЁШӘ Щ…ЫҢвҖҢЪ©ЩҶШҜШӣ overload ШҜЩҲЩ… ЫҢЪ© ProcessedUpdate ШұШ§ Щ…ШіШӘЩӮЫҢЩ…Ш§ЩӢ Щ…ЫҢвҖҢЩҫШ°ЫҢШұШҜ.
    • messages() ЫҢЪ© Ш§ШіЩҶЩҫвҖҢШҙШ§ШӘ ШәЫҢШұЩӮШ§ШЁЩ„ ШӘШәЫҢЫҢШұ ШЁШұЩ…ЫҢвҖҢЪҜШұШҜШ§ЩҶШҜ.
    • clear() ЩҫЫҢШ§Щ…вҖҢЩҮШ§ЫҢ Щ…ЩҶШ·ШЁЩӮ Ш§ШІ ШӘЩ„ЪҜШұШ§Щ… ШӯШ°ЩҒ Щ…ЫҢвҖҢЪ©ЩҶШҜ (ШҜШұ ШҜШіШӘЩҮвҖҢЩҮШ§ЫҢ ЫұЫ°Ы° ШӘШ§ЫҢЫҢ вҖ” Щ…ШӯШҜЩҲШҜЫҢШӘ API deleteMessages) ЩҲ ШўЩҶЩҮШ§ ШұШ§ Ш§ШІ Ш°Ш®ЫҢШұЩҮвҖҢШіШ§ШІЫҢ ЩҫШ§Ъ© Щ…ЫҢвҖҢШіШ§ШІШҜ. Ш°Ш®ЫҢШұЩҮвҖҢШіШ§ШІЫҢ ШөШұЩҒвҖҢЩҶШёШұ Ш§ШІ ЩҶШӘЫҢШ¬ЩҮ ЩҮШұ ШҜШіШӘЩҮ ЩҫШ§Ъ© Щ…ЫҢвҖҢШҙЩҲШҜШҢ ШЁЩҶШ§ШЁШұШ§ЫҢЩҶ Ш®Ш·Ш§ЩҮШ§ЫҢ Щ…ЩҲЩӮШӘ API ШЁШ§Ш№Ш« ЩҶШҙШӘ ЩҲШұЩҲШҜЫҢвҖҢЩҮШ§ ЩҶЩ…ЫҢвҖҢШҙЩҲЩҶШҜ.
    • forget() ЩҒЩӮШ· ЩҲШұЩҲШҜЫҢвҖҢЩҮШ§ ШұШ§ Ш§ШІ Ш°Ш®ЫҢШұЩҮвҖҢШіШ§ШІЫҢ ШӯШ°ЩҒ Щ…ЫҢвҖҢЪ©ЩҶШҜ вҖ” ШӘЩ„ЪҜШұШ§Щ… ШӘШӯШӘвҖҢШӘШЈШ«ЫҢШұ ЩӮШұШ§Шұ ЩҶЩ…ЫҢвҖҢЪҜЫҢШұШҜ.
    • close() Ш§ШҙШӘШұШ§Ъ© Ъ©Щ„ЫҢШҜ ШұШ§ Ш§ШІ ШұШҜЫҢШ§ШЁЫҢ Ш®ЩҲШҜЪ©Ш§Шұ Щ„ШәЩҲ Щ…ЫҢвҖҢЪ©ЩҶШҜ ЩҲ Ш°Ш®ЫҢШұЩҮвҖҢШіШ§ШІЫҢ ШўЩҶ ШұШ§ ЩҫШ§Ъ© Щ…ЫҢвҖҢШіШ§ШІШҜ. Ш§ЫҢЩҶ ЩҶЩ…ЩҲЩҶЩҮ ЩҮЩ…ЪҶЩҶШ§ЩҶ ЩӮШ§ШЁЩ„ Ш§ШіШӘЩҒШ§ШҜЩҮ Ш§ШіШӘШӣ ЩҒШұШ§Ш®ЩҲШ§ЩҶЫҢ ШҜЩҲШЁШ§ШұЩҮ bot.sessions.get(...) ШЁШ§Ш№Ш« Ш§ШҙШӘШұШ§Ъ©вҖҢЪҜШ°Ш§ШұЫҢ Щ…Ш¬ШҜШҜ Щ…ЫҢвҖҢШҙЩҲШҜ.

    Multiple parallel sessionsВ¶

    ШЁШұШ§ЫҢ ЫҢЪ© ЪҶШӘ/Ъ©Ш§ШұШЁШұ ЫҢЪ© ШіШҙЩҶ Щ…ШіШӘЩӮЩ„ ШЁШ§ qualifier Щ…ШҙШ®Шө Ъ©ЩҶЫҢШҜ:

    val wizard  = bot.sessions.get(chat.id, user.id, qualifier = "wizard")
val support = bot.sessions.get(chat.id, user.id, qualifier = "support")

    ШҜШұ ШӘЩҲШ§ШЁШ№ handlerвҖҢШҢ Щ…ЩҲЩ„ШҜ Ъ©ШҜ ktnip ШЁЩҮвҖҢШөЩҲШұШӘ Ш®ЩҲШҜЪ©Ш§Шұ qualifier ЩҮШ§ ШұШ§ Ш§ШІ Ш·ШұЫҢЩӮ annotation @SessionQualifier ШӘЩҶШёЫҢЩ… Щ…ЫҢвҖҢЪ©ЩҶШҜ:

    @CommandHandler(["/help"])
suspend fun help(
    @SessionQualifier("wizard")  wizard:  Session,
    @SessionQualifier("support") support: Session,
    bot: TelegramBot,
) {
    // wizard ЩҲ support ШіШҙЩҶвҖҢЩҮШ§ЫҢ Ш¬ШҜШ§ЪҜШ§ЩҶЩҮвҖҢШ§ЫҢ ШЁШұШ§ЫҢ ЩҮЩ…Ш§ЩҶ chat/user ЩҮШіШӘЩҶШҜ.
}

    ШЁШұШ§ЫҢ ШіШҙЩҶ ЩҫЫҢШҙвҖҢЩҒШұШ¶ (ШЁШҜЩҲЩҶ qualifier) ЩҶЫҢШ§ШІЫҢ ШЁЩҮ annotation ЩҶЫҢШіШӘ.

    Session-aware sendsВ¶

    ШҜЩҲ ШұЩҲШҙ Щ…Ш№Ш§ШҜЩ„ ШЁШұШ§ЫҢ Ш§ШұШіШ§Щ„ ЩҫЫҢШ§Щ… ШҜШ§Ш®Щ„ ШіШҙЩҶ ЩҲШ¬ЩҲШҜ ШҜШ§ШұШҜ вҖ” ЩҮШұ Ъ©ШҜШ§Щ… ШұШ§ Ъ©ЩҮ ШҜШұ Щ…ШӯЩ„ ЩҒШұШ§Ш®ЩҲШ§ЩҶЫҢ ШЁЩҮШӘШұ Щ…ЫҢвҖҢШЁЫҢЩҶШҜЫҢШҜ Ш§ШіШӘЩҒШ§ШҜЩҮ Ъ©ЩҶЫҢШҜ:

    // 1. ЩҫШ§Ші Ъ©ШұШҜЩҶ ШөШұЫҢШӯ ШіШҙЩҶ:
message { "Confirm with yes/no" }.send(user, bot, session)
photo   { "FILE_ID" }.send(chat, bot, session)

// 2. ЫҢШ§ ШЁШ§ШІ Ъ©ШұШҜЩҶ ШЁЩ„ЩҲЪ© ШІЩ…ЫҢЩҶЩҮ ЩҲ ШӯШ°ЩҒ ЩҫШ§ШұШ§Щ…ШӘШұ:
with(session) {
    message { "Confirm with yes/no" }.send(bot)             // ЩҮШҜЩҒ chatId ШіШҙЩҶ
    photo   { "FILE_ID" }.send(to = user, via = bot)
}

    ЩҮШұ ШҜЩҲ Щ…ШіЫҢШұ ЩҫЫҢШ§Щ… ШЁШұЪҜШҙШӘЫҢ ШұШ§ ШЁЩҮвҖҢШ№ЩҶЩҲШ§ЩҶ Direction.Outgoing Ш®ЩҲШҜЪ©Ш§Шұ ШұШҜЫҢШ§ШЁЫҢ Щ…ЫҢвҖҢЪ©ЩҶЩҶШҜ (ЩҶЪҜШ§ЩҮ Ъ©ЩҶЫҢШҜ ШЁЩҮ Action.sendTracked / sendReturningTracked). ЪҶЩҲЩҶ ШіШҙЩҶ ШЁЩҮвҖҢШөЩҲШұШӘ ШөШұЫҢШӯ ЩҫШ§Ші Щ…ЫҢвҖҢШҙЩҲШҜ (ЩҶЩҮ Ш§ШІ Ш·ШұЫҢЩӮ threadвҖ‘ ЫҢШ§ coroutineвҖ‘local) ШұШҜЫҢШ§ШЁЫҢ ЩҮШұЪҜШІ Ш§ШІ ШҜШіШӘ ЩҶЩ…ЫҢвҖҢШұЩҲШҜ ШӯШӘЫҢ ЩҲЩӮШӘЫҢ ЩҮЩҶШҜЩ„ШұЩҮШ§ coroutine ЩҒШұШІЩҶШҜ Ш§Ш¬ШұШ§ Щ…ЫҢвҖҢЪ©ЩҶЩҶШҜ.

    sendReturning(...) ШЁЩҮ ЩҮЩ…Ш§ЩҶ ШҙЪ©Щ„ Ш№Щ…Щ„ Щ…ЫҢвҖҢЪ©ЩҶШҜ: ЩҮШұ Message (ЫҢШ§ Щ„ЫҢШіШӘ ЩҫЫҢШ§Щ…вҖҢЩҮШ§) ШЁШұЪҜШҙШӘЫҢ ЩӮШЁЩ„ Ш§ШІ Ш§ЫҢЩҶ Ъ©ЩҮ Deferred ЩҒШұШ§Ш®ЩҲШ§ЩҶЩҶШҜЩҮ Ъ©Ш§Щ…Щ„ ШҙЩҲШҜШҢ ШҜШұ ШіШҙЩҶ Ш«ШЁШӘ Щ…ЫҢвҖҢШҙЩҲШҜШҢ ШЁЩҶШ§ШЁШұШ§ЫҢЩҶ Щ…ЫҢвҖҢШӘЩҲШ§ЩҶЫҢШҜ ЩҫШ§ШіШ® ШұШ§ ШЁЩҮвҖҢШөЩҲШұШӘ Щ…Ш№Щ…ЩҲЩ„ЫҢ Ш§ШіШӘЩҒШ§ШҜЩҮ Ъ©ЩҶЫҢШҜ.

    Storage backendsВ¶

    SessionStorage ЫҢЪ© Ш§ЫҢЩҶШӘШұЩҒЫҢШі Ъ©ЩҲЪҶЪ© Ш§ШіШӘ:

    interface SessionStorage {
    suspend fun add(key: SessionKey, entry: TrackedMessage)
    suspend fun list(key: SessionKey): List<TrackedMessage>
    suspend fun remove(key: SessionKey, predicate: (TrackedMessage) -> Boolean): Int
    suspend fun clear(key: SessionKey)
}

    ЩҫЫҢШҙвҖҢЩҒШұШ¶ InMemorySessionStorage Ш§ШіШӘ (ЩҫШҙШӘЫҢШЁШ§ЩҶ ConcurrentHashMap). Щ…ЫҢвҖҢШӘЩҲШ§ЩҶЫҢШҜ ШЁШұШ§ЫҢ RedisШҢ JDBC ЩҲ ШәЫҢШұЩҮ ЩҫЫҢШ§ШҜЩҮвҖҢШіШ§ШІЫҢ Ш®ЩҲШҜ ШұШ§ ШЁЩҶЩҲЫҢШіЫҢШҜ ЩҲ Ш§ШІ Ш·ШұЫҢЩӮ ШЁЩ„ЩҲЪ© ЩҫЫҢЪ©ШұШЁЩҶШҜЫҢ ЩҲШөЩ„ Ъ©ЩҶЫҢШҜ.

    ConfigurationВ¶

    val bot = TelegramBot("BOT_TOKEN") {
    sessions {
        keyStrategy   = SessionKeyStrategy.Auto
        storage       = InMemorySessionStorage()
        // managerFactory = SessionManagerFactory { bot, cfg -> CustomSessionManager(bot, cfg) }
    }
}

    ЩҮШұ ШіЩҮ ЩҲЫҢЪҳЪҜЫҢ Щ…ЩӮШҜШ§Шұ ЩҫЫҢШҙвҖҢЩҒШұШ¶ Щ…Ш№ЩӮЩҲЩ„ ШҜШ§ШұЩҶШҜ вҖ” ШЁЩ„ЩҲЪ© sessions { } ЩҒЩӮШ· ШІЩ…Ш§ЩҶЫҢ Щ„Ш§ШІЩ… Ш§ШіШӘ Ъ©ЩҮ ШЁШ®ЩҲШ§ЩҮЫҢШҜ ЪҶЫҢШІЫҢ ШұШ§ ШЁШ§ШІЩҶЩҲЫҢШіЫҢ Ъ©ЩҶЫҢШҜ.

    PerformanceВ¶

    SessionManager.isIdle() ШӘШ§ ШІЩ…Ш§ЩҶЫҢ Ъ©ЩҮ Ш§ЩҲЩ„ЫҢЩҶ ШіШҙЩҶ ШЁШ§ШІ ЩҶШҙЩҲШҜ true Щ…ЫҢвҖҢЩ…Ш§ЩҶШҜ. SessionTrackingInterceptor ШҜШұ ЩҮШұ ШЁЩҮвҖҢШұЩҲШІШұШіШ§ЩҶЫҢ Ш§ЫҢЩҶ Щ…ЩӮШҜШ§Шұ ШұШ§ ШЁШұШұШіЫҢ Щ…ЫҢвҖҢЪ©ЩҶШҜ ЩҲ ЩҲЩӮШӘЫҢ idle ШЁШ§ШҙШҜ Ъ©ЩҲШӘШ§ЩҮ Щ…ЫҢвҖҢШҙЩҲШҜШҢ ШЁЩҶШ§ШЁШұШ§ЫҢЩҶ ШұШЁШ§ШӘвҖҢЩҮШ§ЫҢЫҢ Ъ©ЩҮ ЩҮШұЪҜШІ bot.sessions.get(...) ШұШ§ ШөШҜШ§ ЩҶЩ…ЫҢвҖҢШІЩҶЩҶШҜ ШӘЩҶЩҮШ§ ЫҢЪ© ШЁШұШұШіЫҢ ЩҶЩӮШҙЩҮ ШҜШұ ЩҮШұ ШЁЩҮвҖҢШұЩҲШІШұШіШ§ЩҶЫҢ ЩҫШұШҜШ§Ш®ШӘ Щ…ЫҢвҖҢЪ©ЩҶЩҶШҜ.

    Ш§ШҙШӘШұШ§Ъ©вҖҢЩҮШ§ ШЁШұ ЩҫШ§ЫҢЩҮ predicate ЩҮШіШӘЩҶШҜ: ШЁШ§ШІ Ъ©ШұШҜЩҶ ШіШҙЩҶ ШЁШұШ§ЫҢ ЫҢЪ© Ъ©Щ„ЫҢШҜШҢ predicateвҖҢШ§ЫҢ Ш«ШЁШӘ Щ…ЫҢвҖҢЪ©ЩҶШҜ Ъ©ЩҮ Ш§ЫҢЩҶШӘШұШіЩҫШӘЩҲШұ ШўЩҶ ШұШ§ ШҜШұ ШЁЩҮвҖҢШұЩҲШІШұШіШ§ЩҶЫҢвҖҢЩҮШ§ЫҢ ШЁШ№ШҜЫҢ Щ…ЩӮШ§ЫҢШіЩҮ Щ…ЫҢвҖҢЪ©ЩҶШҜ. session.close() predicate ШұШ§ ШӯШ°ЩҒ Щ…ЫҢвҖҢЪ©ЩҶШҜ.

    Complete example вҖ” ephemeral order flowВ¶

    @CommandHandler(["/order"])
suspend fun startOrder(
    @SessionQualifier("order") order: Session,
    user: User,
    bot: TelegramBot,
) {
    with(order) {
        message { "What would you like to order?" }.send(user, bot)  // auto-tracked
    }
}

@CommandHandler(["/done"])
suspend fun finishOrder(
    @SessionQualifier("order") order: Session,
    user: User,
    bot: TelegramBot,
) {
    // This farewell is sent without the session so it survives clear().
    message { "Order received вҖ” wiping our chat history." }.send(user, bot)

    val removed = order.clear()                   // delete every message in this slice
    order.close()                                  // stop tracking until next /order
    println("Cleared $removed messages for ${user.id}")
}

    ЩҫШ§ШұШ§Щ…ШӘШұ @SessionQualifier("order") Ш§ЫҢЩҶ Ш¬ШұЫҢШ§ЩҶ ШұШ§ Ш§ШІ ЩҮШұ ШіШҙЩҶ ЩҮЩ…ШІЩ…Ш§ЩҶ ШҜЫҢЪҜШұ (ЫҢЪ© wizardШҢ ЫҢЪ© Щ…ЩҲШ¶ЩҲШ№ ЩҫШҙШӘЫҢШЁШ§ЩҶЫҢШҢ вҖҰ) Ъ©ЩҮ ЩҮЩ…Ш§ЩҶ Ъ©Ш§ШұШЁШұ Щ…Щ…Ъ©ЩҶ Ш§ШіШӘ Ш§Ш¬ШұШ§ Ъ©ЩҶШҜШҢ Ш¬ШҜШ§ ЩҶЪҜЩҮ Щ…ЫҢвҖҢШҜШ§ШұШҜ. ЩҮШұ ЩҫШ§ШіШ® Ъ©Ш§ШұШЁШұ ШЁЫҢЩҶ /order ЩҲ /done ШӘЩҲШіШ· Ш§ЫҢЩҶШӘШұШіЩҫШӘЩҲШұ ШЁЩҮвҖҢШөЩҲШұШӘ Ш®ЩҲШҜЪ©Ш§Шұ Ш«ШЁШӘ Щ…ЫҢвҖҢШҙЩҲШҜШӣ ЩҮШұ Ш§ШұШіШ§Щ„ ШұШЁШ§ШӘ ШҜШ§Ш®Щ„ with(order) { вҖҰ } ШЁЩҮвҖҢШөЩҲШұШӘ Ш®ЩҲШҜЪ©Ш§Шұ ШӘЩҲШіШ· overload ШўЪҜШ§ЩҮЫҢвҖҢШҜШ§Шұ ШіШҙЩҶ Ш«ШЁШӘ Щ…ЫҢвҖҢШҙЩҲШҜ.
